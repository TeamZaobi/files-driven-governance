# 反方质询与收敛回路

这个参考件只在“需要制度化反方视角”的时候启用。
不是每个话题都要上。

## 什么时候启用

出现以下情况时再用：

1. 争议会直接影响任务、决策或升级
2. 话题风险高，礼貌讨论不足以暴露问题
3. 团队长期把“没有人反对”误当成“已经收敛”
4. 需要明确记录反对点、答辩和结论

## 最小角色形态

至少有这三个位置：

1. 主张方
2. 质询方
3. 收敛裁定方

一个人可以临时扮演多个位置，但三种功能不能消失。

## 核心回路

1. 提出主张
2. 逐点质询
3. 逐点答辩
4. 判断哪些问题已回答，哪些仍未回答
5. 明确收敛结论：通过、延后、打回、拆题

## 必要产物

至少留下：

1. 争议主题
2. 关键问题列表
3. 每个问题的答辩结果
4. 未解决问题
5. 最终结论和后续动作

## 问题状态语义

建议至少区分：

- 已提出
- 已回答
- 部分回答
- 无法回答
- 已转成下游任务

这样做的目的，是防止“看起来聊过了”替代“真的回答了”。

## 收敛标准

只有在以下条件成立时，才算真正收敛：

1. 关键问题已经逐条处理
2. 结论足以支持下一步动作
3. 残留风险已被明确记录
4. 还没解决的问题有清楚去向

## 需要警惕的失效模式

- 质询变成情绪宣泄
- 答辩只做姿态，不回应问题
- 没有裁定步骤，永远悬而不决
- 结论没有落到任务、决策或回退动作

## 轻量模式

如果话题重要但不至于开完整回路，可以用轻量版：

1. 列三个最强反对点
2. 逐点回应
3. 明确是否继续推进

## 在本技能里的推荐用法

当一个议题会直接影响：

- 规则
- 流程关口
- 角色边界
- 对外展示面

并且争议已经足够实质时，再推荐启用这条回路。
