# 结构家族定位约定

当项目需要为五类核心结构建立稳定读取规则时，使用这个参考件：

1. `policy_or_rules`
2. `object`
3. `workflow`
4. `skill`
5. `agent`

目标是让这些结构在跨工具、跨会话、跨交接时都能稳定找到。

## 每一类都要写清什么

对每一类至少写清：

1. 真源根路径
2. 当前版本锚点
3. 注册表、索引或定位入口
4. 优先读取路径
5. 兜底读取顺序
6. 工具适配入口或摘要入口
7. 直接写入责任边界
8. 红旗信号

## 分家族说明

### 1. `policy_or_rules`

常见真源：

- 总规则文档
- 共享规则目录
- 规则索引

当前版本锚点常见是：

- 当前规则主文档
- 规则 README
- 如果规则按变更对象版本化，则是最近批准的规则变更

推荐读取顺序：

1. 规则索引或当前规则锚点
2. 当前规则主文档
3. 最近批准的规则变更
4. 过程载体或摘要页只用于补上下文

常见适配入口：

- 顶层工具说明文件
- 启动摘要
- 辅助说明文档

红旗信号：

- 工具启动文档变成实际规则来源
- 状态页宣布了一条上游没有批准过的新规则

### 2. `object`

常见真源：

- 对象注册表
- 模式文件
- `OBJECTS.*`
- 对象定义文档

当前版本锚点常见是：

- 对象注册表条目
- 当前对象定义文档
- 单独版本化时最近批准的对象变更

推荐读取顺序：

1. 对象注册表或对象索引
2. 当前对象定义或模式文档
3. 最近批准的对象变更
4. 等对象定义清楚后，再看消费它的流程或技能

常见适配入口：

- 生成类型
- 校验辅助工具
- 渲染后的字段说明

红旗信号：

- 流程文档定义了对象文档没有定义的字段
- 技能为了图省事私造一套对象结构

### 3. `workflow`

常见真源：

- 流程注册表
- `WORKFLOW.*`
- 路径与关口说明

当前版本锚点常见是：

- 流程注册表条目
- 当前流程定义
- 最近批准的流程修订

推荐读取顺序：

1. 流程注册表或生命周期索引
2. 当前流程定义
3. 影响关口和流转的最近批准变更
4. 看板、状态页或脚本只当消费方

常见适配入口：

- 辅助脚本
- 自动化定义
- 状态页面
- 任务板

红旗信号：

- 脚本成了唯一可读的流程定义
- 状态页定义了流程真源从未批准的关口

### 4. `skill`

常见真源：

- 技能注册表
- `skills/src/<skill-name>/`
- `SKILL.md`
- 被技能真实消费的 `references/`

当前版本锚点常见是：

- 技能注册表条目
- 技能包根目录
- `SKILL.md` 及其实际引用的辅助文件

推荐读取顺序：

1. 技能注册表或技能索引
2. 技能包根目录
3. `SKILL.md`
4. 被绑定的参考件、清单、脚本或资源
5. 外部例子只用于对比

常见适配入口：

- 斜杠命令
- CLI 包装
- 提示词包装
- 工具特定启动入口

红旗信号：

- 外部安装器假设替代了本地技能真源
- 游离清单没有绑定进技能包，却被当成技能的一部分

### 5. `agent`

常见真源：

- 角色注册表
- 标准 `AGENT.md`
- 运行时映射表

当前版本锚点常见是：

- 角色注册表条目
- 当前角色约定
- 运行时映射只负责工具或运行时绑定

推荐读取顺序：

1. 角色注册表或角色索引
2. 当前角色约定
3. 角色专属边界说明
4. 运行时映射
5. 工具入口最后再看

常见适配入口：

- 工具侧代理配置
- 运行时映射文件
- 启动提示包装

红旗信号：

- 工具入口成了唯一角色说明
- 角色职责只活在启动提示里

## 跨家族总规则

1. 工具入口默认是适配入口，不是真源
2. 任何家族都应有一个当前版本锚点
3. 同一概念不要依赖多个平级入口同时承担真源职责
4. 读取顺序一旦确定，就尽量跨工具保持稳定
