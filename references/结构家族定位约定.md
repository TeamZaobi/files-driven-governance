# 结构家族定位约定

当项目需要为五类核心结构建立稳定读取规则时，使用这个参考件：

1. `policy_or_rules`
2. `object`
3. `workflow`
4. `skill`
5. `agent`

目标是让这些结构在跨工具、跨会话、跨交接时都能稳定找到。
这份 reference 只负责回答“这些核心结构到哪里找、先锚哪一版”；
不替代场景路由、正式读取顺序、工具适配判断或共享约定。

如果 `workflow` 已进入合同化执行，
或者运行实例、状态页、展示页会被多人、多代理、多工具共享，
定位时不要只盯五类核心结构。
还要顺带判清：

1. 哪些文件属于 `execution_object`
2. 哪些文件属于 `status_projection`
3. 哪些文件属于 `display_projection`

尤其要防止“与 workflow 共置的运行文件”被误认成 workflow 真源。

## 每一类都要写清什么

对每一类至少写清：

1. 真源根路径
2. 当前版本锚点
3. 注册表、索引或定位入口
4. 优先读取路径
5. 兜底读取顺序
6. 工具适配入口或摘要入口
7. 直接写入责任边界
8. 红旗信号

不是每个项目一开始都要把五类全部写满。
只对当前真的承担治理真源或关键控制面的结构家族落这些字段。

## 分家族说明

### 1. `policy_or_rules`

常见真源：

- 总规则文档
- 共享规则目录
- 规则索引
- `rules.contract.json`

当前版本锚点常见是：

- 当前规则主文档
- 规则 README
- 当前规则合同
- 如果规则按变更对象版本化，则是最近批准的规则变更

推荐读取顺序：

1. 规则索引或当前规则锚点
2. 当前规则合同（如果存在）
3. 当前规则主文档
4. 最近批准的规则变更
5. 过程载体或摘要页只用于补上下文

常见适配入口：

- 顶层工具说明文件
- 启动摘要
- 辅助说明文档

红旗信号：

- 工具启动文档变成实际规则来源
- 状态页宣布了一条上游没有批准过的新规则

### 2. `object`

常见真源：

- 对象注册表
- 模式文件
- `*.schema.json`
- `OBJECTS.*`
- 对象定义文档

当前版本锚点常见是：

- 对象注册表条目
- 当前对象定义文档
- 当前对象 schema
- 单独版本化时最近批准的对象变更

推荐读取顺序：

1. 对象注册表或对象索引
2. 当前对象 schema（如果存在）
3. 当前对象定义或模式文档
4. 最近批准的对象变更
5. 等对象定义清楚后，再看消费它的流程或技能

常见适配入口：

- 生成类型
- 校验辅助工具
- 渲染后的字段说明

红旗信号：

- 流程文档定义了对象文档没有定义的字段
- 技能为了图省事私造一套对象结构

### 3. `workflow`

常见真源：

- 流程注册表
- `WORKFLOW.*`
- `workflow.contract.json`
- 路径与关口说明

当前版本锚点常见是：

- 流程注册表条目
- 当前流程定义
- 当前 workflow 合同
- 最近批准的流程修订

推荐读取顺序：

1. 流程注册表或生命周期索引
2. 当前 workflow 合同
3. 当前流程定义
4. 影响关口和流转的最近批准变更
5. 看板、状态页或脚本只当消费方

常见适配入口：

- 辅助脚本
- 自动化定义
- 状态页面
- 任务板

红旗信号：

- 脚本成了唯一可读的流程定义
- `WORKFLOW.md` 与 `workflow.contract.json` 各自定义了不同的状态和关口
- 状态页定义了流程真源从未批准的关口

### 4. `skill`

常见真源：

- 技能注册表
- `skills/src/<skill-name>/`
- `SKILL.md`
- `skill.manifest.json`
- 被技能真实消费的 `references/`

当前版本锚点常见是：

- 技能注册表条目
- 技能包根目录
- `SKILL.md` 及其实际引用的辅助文件
- 当前技能 manifest

推荐读取顺序：

1. 技能注册表或技能索引
2. 技能包根目录
3. 当前技能 manifest（如果存在）
4. `SKILL.md`
5. 被绑定的参考件、清单、脚本或资源
6. 外部例子只用于对比

常见适配入口：

- 斜杠命令
- CLI 包装
- 提示词包装
- 工具特定启动入口

红旗信号：

- 外部安装器假设替代了本地技能真源
- 游离清单没有绑定进技能包，却被当成技能的一部分

### 5. `agent`

常见真源：

- 角色注册表
- 标准 `AGENT.md`
- `agent.contract.json`
- 运行时映射表

当前版本锚点常见是：

- 角色注册表条目
- 当前角色约定
- 当前角色合同
- 运行时映射只负责工具或运行时绑定

推荐读取顺序：

1. 角色注册表或角色索引
2. 当前角色合同（如果存在）
3. 当前角色约定
4. 角色专属边界说明
5. 运行时映射
6. 工具入口最后再看

常见适配入口：

- 工具侧代理配置
- 运行时映射文件
- 启动提示包装

红旗信号：

- 工具入口成了唯一角色说明
- `AGENT.md` 与 `agent.contract.json` 对批准边界给出不同说法
- 角色职责只活在启动提示里

## 条件扩展：运行实例与投影家族

当 `workflow` 已进入受控模式，
或者运行态资产会被多人、多代理、多工具共同消费时，
再额外定位以下三类结构。

### 6. `execution_object`

常见真源：

- `workflow.state.json`
- `workflow.events.jsonl`
- `decision_package`
- `validation_record`

当前版本锚点常见是：

- 当前活跃运行实例
- 当前事件流或最近一次关键决策记录
- 当前有效的验证记录

推荐读取顺序：

1. 当前运行实例
2. 当前事件流或关键决策记录
3. 与本轮判断直接相关的验证记录
4. 需要压缩过程时，再看 `process_projection` 或状态页

红旗信号：

- 运行实例反向改写上游 workflow 合同
- 与 workflow 共置的实例文件被当成 workflow 真源
- 项目只有状态页，没有可追溯的实例或事件流

### 7. `status_projection`

常见落点：

- 当前状态页
- handoff 包
- process projection
- 恢复摘要页

当前版本锚点常见是：

- 当前状态摘要
- 最近一次交接摘要
- 当前恢复入口

推荐读取顺序：

1. 先确认上游 workflow 合同和 execution_object 已大致可信
2. 再读当前状态摘要或 handoff 包
3. 发现状态、关口或下一步与上游冲突时，立即回上游裁决
4. 工具侧状态页最后再看

红旗信号：

- 状态页自己定义新的 `allowed_next_steps`
- 状态页给出上游从未批准的放行结论
- 不同工具显示的当前状态互相冲突

如果项目需要结构化的当前状态入口，
建议最小落成 `status.projection.json`。
它只应持有：

- `current_node_id`
- `gate_state`
- `missing_evidence_refs`
- `forbidden_output_refs`
- `summary`
- 来源锚点：`source_last_event_id`
- 生成时间：`generated_at`

不要让它持有：

- `allowed_next_step_refs`
- `approval_ref`
- `guard_policy_refs`
- 新造的 `next_step`
- 新的放行结论

### 8. `display_projection`

常见落点：

- 汇报页
- 对外说明页
- deck、报告或公告
- 展示型仪表盘

当前版本锚点常见是：

- 当前对外版本
- 当前汇报版本
- 当前展示入口

推荐读取顺序：

1. 先确认上游 truth、workflow 和当前状态都已可信
2. 再读当前展示版本
3. 需要核真时，回到上游合同、实例和状态摘要

红旗信号：

- 展示页成了别人理解项目状态的唯一入口
- 展示页替上游定义新状态、新关口或新权限
- 对外版本比内部状态页更新得更快

## 跨家族总规则

1. 工具程序和入口默认是适配入口，不是真源；仓库中的项目级规则文件按职责归入 `agent / skill / policy_or_rules`
2. 任何家族都应有一个当前版本锚点
3. 同一概念不要依赖多个平级入口同时承担真源职责
4. 读取顺序一旦确定，就尽量跨工具保持稳定
5. 如果某个家族已经引入结构化合同，合同负责机读控制语义，解释文档负责人读说明，两者不能互相抢真源职责
6. 共置只等于方便查找，不等于家族归属或写权转移
7. `workflow.state.json`、`workflow.events.jsonl` 默认归 `execution_object`，即使它们和 workflow 合同放在同一目录

定位清楚后，需要固定顺序时读 [官方读取顺序](官方读取顺序.md)。
工具入口遮住真源时，读 [工具适配对照表](工具适配对照表.md)。
多人、多代理或多工具共享同一事实时，读 [跨层共享约定](跨层共享约定.md)。
