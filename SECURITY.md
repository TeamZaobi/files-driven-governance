# 安全说明

本仓库不是线上服务，但它会影响项目治理、权限边界和多工具协作设计，因此安全问题仍然重要。

## 报告范围

欢迎报告以下问题：

- 技能内容鼓励不安全的权限设计
- 工具入口被错误提升为真源
- 多工具协作中出现高风险的信息泄露路径
- 跨层共享约定遗漏关键写入或审批边界
- 文档建议明显削弱复核、回退或审计能力

## 不在本范围内

以下内容通常不属于安全漏洞：

- 一般性的写作偏好
- 非安全性的文案风格问题
- 对治理强度的偏好分歧

## 报告方式

如果你准备公开发布到 GitHub，建议先通过私下渠道同步给维护者，再决定是否公开建 Issue。

## 处理原则

1. 先确认问题是否影响权限、审计、回退或数据泄露边界
2. 再判断是修改 `SKILL.md` 还是修改 `references/`
3. 如果问题会影响多工具环境，优先修正跨工具和跨层共享约定
