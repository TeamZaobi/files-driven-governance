# hosted-knowledge-governance

This is an official fixture for host-name-first governance triage.
It does not teach how to operate Obsidian, Notion, Docs, Sheets, or Slides.
It only shows how `files-driven` should classify host surfaces when the real question is about truth source, write authority, projection, drift, or recovery.

Use this fixture when you want to test questions like:

1. Which page counts as the current source of truth
2. Which pages are only index, status, dashboard, or display surfaces
3. Which host objects can be edited directly, and which must route back upstream
4. When a host-named request should stay with `files-driven`, and when it should be handed back to a host-specific skill

Companion asset:

- [`classification.expected.json`](./classification.expected.json)

The sample files intentionally mix:

- `Obsidian` project spec, `MOC`, status, and `Canvas`
- `Notion` rules home and database view
- `Google Docs` spec
- `Google Sheets` dashboard
- `Google Slides` weekly review deck

This fixture is not a governed pack.
It is a machine-checked example set for hosted knowledge-work routing and authority triage.
