# issue_ledger

## 作用

把 discussion 里还没收掉的问题，改写成可追踪的 issue 列表。

## 示例内容

- issue-01: 当前范围不够清楚
- issue-02: 关键证据还没有补齐
- issue-03: 需要一个明确 owner 来裁定下一步

## 记录原则

- 每个 issue 都要能追溯回 discussion
- 每个 issue 都要说明影响面和状态
- 不要把 issue ledger 继续写成聊天纪要

## 什么时候晋升

- 当 issue 已经足够稳定，可以压成一个 `decision_package`
- 当 issue 已经只剩执行归属，可以落到 `task`
