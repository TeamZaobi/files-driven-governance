# adversarial-convergence

This is a minimal official example for the `adversarial_inquiry -> defense -> convergence` path.
It keeps the disputed claim explicit, answers objections point by point, and only closes once the remaining gaps are named.

## 主张

- 当同一问题出现反方质询时，不先追求“看起来一致”，而是先把争议点拆开，再按证据逐点答辩，最后再做收敛裁定。
- 收敛的目标不是消灭分歧，而是明确哪些点已稳定、哪些点仍未决、哪些点需要重新打开。

## 问题清单

1. 这是不是同一个事实问题，还是不同定义被混在了一起？
2. 现有证据是否足以支持主张，还是还缺上游材料？
3. 哪些质询属于事实核对，哪些属于边界澄清，哪些属于流程约束？
4. 哪些点即使答辩后仍不能收敛？

## 逐点答辩

- 对问题 1：先确认共同上游事实，再区分“事实分歧”与“概念分歧”，避免把不同层次的争议合并处理。
- 对问题 2：只接受可追溯到上游材料的回答；如果证据不足，就明确标记为未完成，而不是提前裁掉争议。
- 对问题 3：事实核对用证据回答，边界澄清用定义回答，流程约束用规则回答，三者不要混写。
- 对问题 4：如果某点仍依赖缺失证据、未冻结定义或未确认责任人，就保留未决状态，不进入假收敛。

## 未决点

- 还有哪些证据没有补齐。
- 哪些术语仍然需要统一定义。
- 哪些结论必须等到后续轮次才能裁定。

## 收敛裁定

- 已确认的部分可以收敛成当前版本结论。
- 未决点保持显式挂起，不伪装成共识。
- 如果新的上游证据改变了前提，允许重新打开同一轮质询。
- 这份裁定只针对当前轮次，不把临时结论升级成永久真源。

## Non-goals

- 不绑定任何源项目私货。
- 不引入额外角色矩阵或工具约束。
- 不把答辩结果写成新的控制真源。
