# 项目治理能力模型 v1（历史阶段入口）

这份文档不再与当前底层模型竞争真源地位。
现在的唯一底层真源是 [项目治理能力模型](./项目治理能力模型.md)。

## 1. `v1` 在当前体系里的位置

`v1` 没有被推翻。
它已经作为基础层并入统一真源，主要保留了这些贡献：

1. 把项目治理从“依赖 prose 和聊天习惯”推进到“以文件承载真源、合同、实例与检查执行面”
2. 固定 `truth_source / execution_object / status_projection / display_projection` 四层分层
3. 保留系统、信息、控制三视角和八类结构家族
4. 把高风险 `workflow` 推向最小合同化、实例化和验证链

如果你过去沿着 `v1` 路径工作，现在应把它理解为统一模型的基础底座，
而不是一份独立于 `v2 / v2.1` 的并列版本。
