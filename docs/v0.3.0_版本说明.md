# v0.3.0 版本说明

发布日期：`2026-04-07`

## 本次版本做了什么

`v0.3.0` 把这套技能从“结构治理说明”推进成“项目治理能力模型 v1”。

这一版不只是补几份参考件，而是把受控流程的最小合同、运行实例、校验链路和入口文档一起收口成了可落地的一组公开资产。

## 主要变化

1. 新增 [docs/项目治理能力模型_v1.md](./项目治理能力模型_v1.md)，把重构对象正式上提为“项目治理设计能力”
2. 新增 [schemas/README.md](../schemas/README.md) 与一组最小 schema 草案，把高风险控制语义收成机读入口
3. 新增 [examples/smoke-governed-review](../examples/smoke-governed-review) 最小 smoke 资产包，提供可跑的受控流程示例
4. 新增 [QUICKSTART.md](../QUICKSTART.md) 与 [MIGRATION.md](../MIGRATION.md)，把最小上手和旧约定迁移说明抬到入口层
5. 新增 [tests/test_validate_governance_assets.py](../tests/test_validate_governance_assets.py) 与 [governance-assets-ci.yml](../.github/workflows/governance-assets-ci.yml)，把最小回归和 CI 接进仓库
6. 收紧 [docs/语言体系规范.md](./语言体系规范.md) 与 [README.md](../README.md) 的表达边界，把入口层改回中文主叙事

## 为什么这次版本重要

因为从这一版开始，`files-driven` 不再只是告诉你“该怎么理解结构”，而开始提供一套最小可执行的治理资产：

- 哪些文件定义流程控制
- 哪些文件定义对象、证据和批准对象
- 哪些文件记录运行实例
- 哪些文件只能做状态摘要
- 校验脚本和最小 CI 如何把这些边界守住

这让它从“方法说明”更进一步变成“可验证的项目治理能力”。

## 推荐发布标题

`v0.3.0 - 项目治理能力模型 v1`
