# 流程工具方法经验全景盘点

## 用途

这份文档用于把 `files-driven` 相关的流程、工具、方法和经验一次性盘清。
它不是新的本体定义，而是一个全景台账入口，帮助团队回答：

1. 现在已经有哪些可用资产
2. 这些资产分别属于流程、工具、方法还是经验
3. 哪些已经稳定可复用，哪些还只是草案或试运行

## 四个盘点面

### 1. 流程

关注“动作怎么走”：

- `install / register / repair / audit`
- starter 冷启动
- scaffold 校验
- runtime 观察、候选保留与能力晋升
- self-hosting / AI-Native E2E 的回放验证

### 2. 工具

关注“靠什么执行”：

- `scripts/bootstrap_files_engine_starter.py`
- `scripts/manage_files_engine.py`
- `scripts/validate_files_engine_scaffold.py`
- pack / runtime / governance / adoption 对应的 checker、runner 和回归
- 宿主原生 workflow、subagents、approvals

### 3. 方法

关注“用什么判断”：

- 真源与投影分层
- 入口规则、能力规则、项目规则、项目实体四分法
- `capability_scope / project_scope / runtime_scope`
- 控制强度判断
- discussion 收口、候选保留、能力晋升

### 4. 经验

关注“过去踩过什么坑、现在怎样避免”：

- README 被写成真源
- metadata、手册、说明书口径彼此漂移
- 入口层偷带本体定义
- 规则直接热改，不经过候选层
- 宿主品牌名先行，掩盖了真正的治理问题
