# GitHub 仓库元数据建议

## 1. 建议仓库名

推荐仓库名：

`files-driven-governance`

如果希望和当前技能目录保持一致，也可以继续使用：

`files-driven`

## 2. 仓库简介区文案

推荐短描述：

`文档驱动的项目治理技能，用说人话的需求确认、结构家族划分和可迁移的意图触发约定，为 AI Agent 与多工具项目设计合适的文档治理结构。`

如果平台必须填写英文短描述，可用：

`Project-structure governance skill for AI agent systems, built around plain-language boundary confirmation, document-based source management, and portable trigger intents.`

## 3. 主题标签建议

推荐主题标签：

- `ai-agents`
- `project-governance`
- `documentation`
- `agentic-workflows`
- `multi-agent`
- `knowledge-management`
- `kanban`
- `spec-driven-development`
- `openclaw`
- `skills`

## 4. 社交预览区文案建议

标题建议：

`文档驱动的项目治理`

副标题建议：

`用文档而不是聊天习惯来稳住规则、流程、角色和当前状态`

说明文案建议：

`先确认边界，再划分结构家族，再设计真源、共享约定和恢复链，让 AI Agent 项目在多人、多代理、多工具环境下也能保持可读、可交接、可回退。`

## 5. 当前发布版本建议

推荐 tag：

`v0.2.7`

推荐发布标题：

`v0.2.7 - 意图触发约定`

推荐发布说明：

```text
v0.2.7 把“继续开发”“开始审计”“反思”“推进”这类短口令，正式提升为可迁移的上游约定。

本次版本包括：
- 新增意图触发约定参考件
- 区分主意图、别名层和修饰项层
- 在输出约定里增加短口令治理区块
- 同步更新说明书、元数据和发布文案

适用于 AI Agent 项目、OpenClaw 类项目以及多工具协作环境。
```

## 6. 许可证建议

当前仓库默认使用：

`MIT`

如果后续想切换为：

- `Apache-2.0`
- `BSD-3-Clause`
- 商业闭源许可

建议同步修改：

- `LICENSE`
- `README.md`
- GitHub About
- Release 文案

## 7. 讨论区与议题区设置建议

建议启用：

- 议题区（Issues）
- 合并请求（Pull Requests）

可选启用：

- 讨论区（Discussions）

通常不建议启用 Wiki，除非你计划把外部知识库也直接放进 GitHub 里维护。
