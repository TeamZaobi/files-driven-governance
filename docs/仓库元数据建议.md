# GitHub 仓库元数据建议

## 1. 建议仓库名

推荐仓库名：

`files-driven`

历史上如果已经用过：

`files-driven-governance`

可作为迁移兼容名保留，但不再作为优先推荐名。

## 2. 仓库简介区文案

推荐短描述：

`files-driven：先按动作、scope 与一级关口判断，再补四层、结构家族、共享约定和恢复链，帮助 AI Agent 与多工具项目稳住真源与写权。`

如果平台必须填写英文短描述，可用：

`files-driven for AI agent systems: stabilize shared truth, write boundaries, and recovery paths across people, agents, and tools.`

## 3. 主题标签建议

推荐主题标签：

- `ai-agents`
- `project-governance`
- `documentation`
- `agentic-workflows`
- `multi-agent`
- `knowledge-management`
- `kanban`
- `spec-driven-development`

## 4. 社交预览区文案建议

标题建议：

`files-driven`

副标题建议：

`用文档而不是聊天习惯来稳住规则、流程、角色和当前状态`

说明文案建议：

`先按动作、scope 与一级关口稳住入口，再补四层、结构家族、共享约定和恢复链，让 AI Agent 项目在多人、多代理、多工具环境下也能保持可读、可交接、可回退。`

## 5. 当前发布版本建议

推荐 tag：

`v0.5.0`

推荐发布标题：

`v0.5.0 - 受控推进 Runner 与体检入口收口`

推荐发布说明：

```text
v0.5.0 不再继续扩新世界观，而是把当前 tranche 的受控推进能力与采用入口正式收口成一个小版本发布。

本次版本包括：
- 新增 script-controlled repo treatment rollout runner，可把既定推进方案编码成 governed run pack，并调度 Codex CLI 受控落地
- 正式把用户侧入口翻译压成 `体检 -> 诊断 -> 治疗 -> 复查 -> 随访`，但不污染底层 canonical 合同语义
- 明确 `audit` 当前默认是脚手架基础体检，不再把它包装成对下游项目的全量系统体检
- 补齐当前 tranche 的发布锚点、版本说明和维护入口，让 self-hosting 推进不再只停在讨论层

适用于已经进入 `files-driven` 主线、并希望把它从“已有执行闭环”继续推进到“可受控收口、可正式发布、对采用者更说人话”的治理场景。
```

## 6. 许可证建议

当前仓库默认使用：

`MIT`

如果后续想切换为：

- `Apache-2.0`
- `BSD-3-Clause`
- 商业闭源许可

建议同步修改：

- `LICENSE`
- `README.md`
- GitHub About
- Release 文案

## 7. 讨论区与议题区设置建议

建议启用：

- 议题区（Issues）
- 合并请求（Pull Requests）

可选启用：

- 讨论区（Discussions）

通常不建议启用 Wiki，除非你计划把外部知识库也直接放进 GitHub 里维护。
