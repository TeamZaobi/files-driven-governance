# v0.5.1 版本说明

发布日期：`2026-04-15`

## 本次版本做了什么

`v0.5.1` 是一次口径收紧补丁版，不扩世界观。
它重点解决一个执行问题：

**下游项目的 E2E 测试目标经常“方向对，但不够直达”。**

这一版把 E2E 通过线改成可执行协议，避免继续停在原则表述。

## 主要变化

1. 调整 [AI-Native 与 Skill 驱动 E2E 验收矩阵](./AI-Native与Skill驱动E2E验收矩阵.md)，新增“测试目标直达协议（下游默认）”。
2. 冻结 E2E 通过线：
   - `真实入口触发`
   - `真源读取正确`
   - `写权受控执行`
   - `冷启动可恢复重放`
   - `oracle 一致`
3. 统一下游默认八字段：
   - `entrypoint`
   - `truth_source_allowlist`
   - `projection_denylist`
   - `write_allowlist`
   - `write_role`
   - `oracle`
   - `replay_seed`
   - `failure_boundary`
4. 调整 [SKILL.md](../SKILL.md)，把上述协议写入 E2E 路由入口，确保入口文案与验收矩阵一致。
5. 调整 [起步阶段：故事与测试对齐](../references/起步阶段_故事与测试对齐.md)，补“把测试目标写成直达句”的最小模板。
6. 调整 [tests/test_end_to_end_governance_alignment.py](../tests/test_end_to_end_governance_alignment.py)，把这套协议关键词冻结进回归，防止口径回退。

## 为什么这次版本重要

之前的 E2E 叙事已经能覆盖 lane，但下游执行时仍容易出现两类偏差：

1. 有路径覆盖，没有明确 oracle
2. 有测试描述，没有 replay/写权边界

`v0.5.1` 的价值是把“可理解”升级成“可判定、可重放、可审计”。

## 这一版不代表什么

`v0.5.1` 不是新 tranche，也不改变 `v1 -> v2 -> v2.1` 的能力模型。
它只是把当前 E2E 验收口径从说明层推进到执行层约束。

## 推荐发布标题

`v0.5.1 - E2E 目标直达协议与验收口径收紧`
