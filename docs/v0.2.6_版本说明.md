# v0.2.6 版本说明

发布日期：`2026-03-25`

## 本次补丁做了什么

`v0.2.6` 把“先确认边界”进一步升级成一套可直接执行的说人话需求确认工具包。

## 主要变化

1. 新增 [references/说人话需求确认工具包.md](../references/说人话需求确认工具包.md)
2. `SKILL.md` 要求提完问题后先起草故事和测试，再进入治理设计
3. 输出约定开始要求边界未稳时先输出问题、故事草案、测试草案、非目标和验收责任人
4. 起步确认与把握度参考件同步接入这套工具包

## 为什么这次补丁重要

因为“知道该先确认”不等于“知道该怎么确认、确认到什么程度才算够”。
工具包把这个动作固定下来，降低了不同操作者之间的偏差。

## 推荐发布标题

`v0.2.6 - 说人话需求确认工具包`
