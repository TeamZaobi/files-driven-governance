# v0.2.5 版本说明

发布日期：`2026-03-25`

## 本次补丁做了什么

`v0.2.5` 把“方向与边界锚点”正式前置，要求先确认首批交付物和验收边界，再展开治理设计。

## 主要变化

1. 新增 [references/起步阶段_故事与测试对齐.md](../references/起步阶段_故事与测试对齐.md)
2. `SKILL.md` 开始要求起步阶段先确认使用场景、用户故事、测试用例和非目标
3. 输出约定和场景手册同步把起步边界确认前置

## 为什么这次补丁重要

因为早期项目最容易在“看起来只差一点点”的边界漂移里，把后续结构和开发范围一起带偏。

## 推荐发布标题

`v0.2.5 - 方向与边界锚点`
