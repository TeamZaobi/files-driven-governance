# v0.3.1 版本说明

发布日期：`2026-04-10`

## 本次补丁做了什么

`v0.3.1` 没有重开 `v0.3.0` 的能力边界，
而是把 `v0.3.0` 之后已经发生的两条整理线正式收口成一个补丁版：

1. 把 `AI-Native` 同构团队协作前提补成公开 reference
2. 把项目本身与 smoke pack 的用户故事、测试用例和验收责任人补成显式资产，并把最小检查接进回归

## 主要变化

1. 新增 [references/AI-Native同构团队协作.md](../references/AI-Native同构团队协作.md)，明确当团队默认通过 AI 工具工作、项目级规则文件跟仓共享时，应先按“多个执行上下文共享同一 Git 存储”理解协作。
2. 新增 [PROJECT_STORIES_AND_TESTS.md](../PROJECT_STORIES_AND_TESTS.md) 与 [tests/test_project_stories_and_tests.py](../tests/test_project_stories_and_tests.py)，把本仓库自己当前迭代的具体用户故事、测试用例、非目标和验收责任人固定成项目级入口。
3. 新增 [examples/smoke-governed-review/BOUNDARY.md](../examples/smoke-governed-review/BOUNDARY.md)，把 smoke pack 的场景、故事、测试和验收责任人显式落成 pack 级入口。
4. 扩展 [scripts/validate_governance_assets.py](../scripts/validate_governance_assets.py) 与 [tests/test_validate_governance_assets.py](../tests/test_validate_governance_assets.py)，开始对 `BOUNDARY.md` 执行最小结构和失败边界检查。
5. 调整 [README.md](../README.md)、[QUICKSTART.md](../QUICKSTART.md)、[MIGRATION.md](../MIGRATION.md)、[SKILL.md](../SKILL.md) 与相关 references，统一故事/测试数量口径，收窄首次阅读路径，并减少入口溢出、规则漂移和过度设计。
6. 补齐 [docs/v0.3.0_版本说明.md](./v0.3.0_版本说明.md)，让 `v0.3.0` 的历史说明和当前补丁发布链完整接上。

## 为什么这次补丁重要

因为 `v0.3.0` 已经把仓库推进到“项目治理能力模型 v1”，
接下来最容易失稳的地方，不是 schema 或 validator 不存在，
而是：

- 协作前提理解不一致
- 项目本身缺少可直接维护的故事与测试入口
- pack 级示例只有合同，没有显式边界入口
- 入口文档、说明文档和回归链开始彼此漂移

`v0.3.1` 的价值，就是把这些会直接影响长期 AI Coding 可维护性的缺口，补成最小但正式的发布资产。
